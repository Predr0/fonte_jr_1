-Add, adicionar novos arquivos a pasta usada para o repositório e depois ser salvo na nuvem.

-Status, verificar o status dos arquivos na pasta do repositório e inspecionar quais arquivos foram adicionados e quais não foram.

-Commit, é quando você salva um estado do seu repositório naquele momento, não sendo alterado, a não ser que você especificamente queira isso.

-Push, comando para enviar o estado atual do repositório para a nuvem, no Github.

-Pull, comando para atualizar o repositório e buscar a versão mais atual para todos os conteúdos.

-Pull request, quando se trabalha com uma equipe e quer fazer um merge dos códigos, para ser solicitada uma aprovação pela equipe ou por pessoas específicas da equipe.

-Clone, Copiar um repositório de um git para um outro repositório.
